# F1 Style and Claim Rules

1. Do not use audit numbers in manuscript section files.
2. Attribute classical results where they first appear.
3. Label MF-R036 as a lemma, MF-R047 as background algebra, MF-R048 as an
   exact framework corollary, and MF-R049 as the central exact counterexample.
4. Use “provisional informational architecture,” never “unique minimal
   ontology.”
5. Use “formal exponential effective score,” not physical free energy.
6. Do not use “emergent spacetime,” “derived gravity,” or equivalent language.
7. Distinguish static sufficiency from dynamic autonomy.
8. State assumptions before every theorem or proposition.
9. Keep computational audits in provenance/data availability, not in the
   mathematical narrative.
10. Write the abstract only after the complete body exists.

11. State explicitly that IIA alone does not select the exponential family.
12. Distinguish raw-scale degeneracy from standardized-parameter
    identifiability.
13. Treat maximum entropy as conditional on a supplied reference measure and
    constraint.
14. Use \(eta\) for a raw-score kernel when useful and \(\lambda\) for the
    transform/standardized family, explaining any identification.
